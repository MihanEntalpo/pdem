Описание работы демона pdem:
1) конфиг хранится в ~/.config/gambas/pdem.conf
2) порт подключения по умолчанию = 5555
3) принцип работы: демон запускается и ждет коннектов на свой порт. После подключения
ему можно слать команды, на которые он будет слать ответы.

Протокол таков:
отправляемое сообщение должно иметь вид:
[CMD[команда]CMD] 
команда - это набор слов разделенных пробелом. Из них первое - это наименование команды, а
остальные - это параметры. Если в одном из параметров требуется вписать пробел, можно
экранировать его слэшем, вот так "\ " (без кавычек). Чтобы написать слеш перед пробелом так, 
чтобы он не считался экранирующим символом - нужно заэкранировать его самого: "\\ " - эта
строка будет распознана как слеш и пробел как знак конца данного параметра и начала следующего.
Длина команды теоретически не ограничена. Практически слишком большой она и не должна быть.
Ответ на команду формируется так:
[ANS[текст ответа]ANS]

Команды, которые поддерживает демон:
-> help, без параметров - выводит описание с примерами всех команда
-> show cmd param1 param2 param3 - команда show отображает результат парсинга остальной строки
так, как будто именно эта строка и была передана (без show), это может пригодиться для отладки
-> runprocess name title type command - запускает процесс.
    параметры:
            name - это название по которому процесс будет идентифицироваться. Должно быть уникальным
                       среди всех запущеных процессов
            title    - заголовок, или "красивое" название процесса, которое может носить описательную
                        функцию.
           type    - тип процесса. Сейчас поддерживаются два local и http - первый выполняет любую
                        консольную команду, а второй - делает http-GET-запрос
           command - собственно консольная команда, или URL, по которому нужно отправить запрос
-> proclist, без параметров - выводит список всех работающих процессов. Каждому процессу отдлельная
           строка, каждый параметр процесса отделен от других пробелом по тем же принципам, по 
           которым парсится строка с командами. Выводится имя, заголовок, тип, команда процесса, а также, время выполнения,
           прогресс, приблизительное время выполнения, и все пользовательские переменные процесса (об это далее)
-> kill procname1 procname2 procname3 - убивает процессы, имена которых переданы. Передать можно сколько угодно имен.
-> disconnect, без параметров - отключает клиента от сервера
-> disconnectall, без параметров - отключает всех клиентов
-> stop, без параметров - останавливает демон

Как процессы, запускаемые демоном могут взаимодействовать с ним:
Если процесс является консольным, то он может отправлять сообщения демону путем вывода их на свой стандартный вывод.
Если процесс - http-get, то он может отправлять команды просто выводя их в документе (при этом, если это, например PHP, нужно 
спользовать инструкцию implicit_flush(), и не пользоваться буферами и кэшем выводимой информации.
Сообщение от процесса к демону выглядит так:
[PDEM[сообщение]PDEM]
сообщения бывают 3-х видов:
1) [PDEM[progressenabled]PDEM] - это сообщение говорит демону о том, что данный процесс умеет подсчитывать свой прогресс выполнения
2) [PDEM[progress=10]PDEM] - это сообщение содержит занчения процента выполнения процесса. Число от 0 до 100. Любое
другое будет приведено к данным границам.
3) [PDEM[var:x=10]PDEM] - это сообщение содержит значение пользовательской переменной процесса, где x - это имя, а 10-
значение (в общем случае и то и другое - просто строка символов, поэтому допустима такая запись [PDEM[var:$%#$=ЖЖЖ]PDEM] - 
она означает создание переменной с именем $%#$ и значением ЖЖЖ. Может использоваться для чего угодно, например, для
вывода [PDEM[var:pics_converted=156]PDEM] - количество сконвертированных скриптом картинок, ну и т.д.
